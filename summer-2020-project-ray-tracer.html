Summer 2020 project: Raytracing!

15 Aug 2020
  • C++
  • raytracing

This summer I have implemented a basic Raytracer as my little project for one week, in order to improve my knowledge on C++ language, start this blog and show my programming skills. Also, I have to say that I’m interested in the Computer Graphics branch of Computer Science.

Ray tracing is a technique that consists in tracing a path from an imaginary eye through each pixel in a virtual screen and calculating the color of the object visible through it. So in the end an image of a three-dimensional scene is generated. This technique is used in the film industry to generate film visual effects (VFX).

So, in order to do the project I have followed the book Ray Tracing in One Weekend from Peter Shirley, that explains in an easy way, how to program a ray tracer without going deeper in the math behind it, making it easier to understand the process to generate an image. In addition, I have checked out the page Scratch a pixel in order to go deeper in some parts.

In summary, I have done the next steps:

  • Create the classes for representing rays, vectors, materials, objects, the camera…
  • Added some objects, in this case spheres, with different colors and materials.
  • Added shading using the surface normals that are calculated when a ray hits an object.
  • Implemented an anti-aliasing technique which, for a given pixel, the program randomly takes several samples of the pixel and sends rays through these samples. The colors of these rays are then averaged. With this computation, the edges of the objects are blended.
  • Implemented different materials: diffuse, metal and dielectrics.
  • A positionable camera.

I will continue with this project adding more objects (like planes, cubes…), adding lights and reading the next book.
