CliWidget first release

25 Sep 2020
  • C++
  • library
  • widget

This month I have been working on a project in order to put in practice the knowledge that I acquired during the lockdown. I didn’t want to do the typical project to print the lines of a ticket, implement the management of a library and the users that borrow books or other typical examples of beginner projects. I was thinking about make something more useful and then, after some time, I thought about the idea of making a library of UI widgets for the terminal.

CliWidget, what is this?

CliWidget is a library that comes with some widgets to help developers to make better UI’s for their terminal programs and focus on their development instead of making them the UI. This library comes with four widgets for the moment:

  • Text: A simple text class for displaying text in the terminal. Additionally, you can add some format to the text like underline, bold, blink, foreground color, background color…

  • InputPassword: A class that creates a password input. The input works like when a password is introduced in Linux terminal: it hides the user input from the terminal so others cannot view what the user is typing. Also you can add a regular expression (regex) and call the function validate() to validate if the password passes the regex.

  • Select: This class is designed as a menu. The options (a vector of strings) are passed through the constructor and then, when the menu is displayed, the user can select one option of all.

  • MultiSelect: This class is like the Select class, but the difference is that users can choose none or more than one option, selecting or deselecting them with the space bar.

How does it works?

The library makes use of the ANSI scape sequences to add the format to the texts and to display background colors and foreground colors. Also it executes the commands stty and tput for hiding the input and the cursor of the terminal respectively and later reset it to its original state.

In the future I will add a root class that will have the common attributes and functions and the widgets classes will inherit from it.

Future improvements

For the moment, the library only works in Linux, I haven’t tested it in Windows OS and MacOS. So the next steps are to make it portable to other platforms and correct bugs that are or will appear or that are reported.

Meanwhile, I will be thinking of other possible widgets that can help other developers.
